About the Application

  • Model: Random Forest Regression trained on official SCRB crime dataset
  • Inputs: Year, District, Crime Category
  • Output: Predicted crime rate, estimated case count, and risk classification
  • Data Pipeline: Data cleaning → Feature engineering → Model training → Deployment
  • Evaluation: Cross-validation with consistent performance across districts
  • Deployment: Flask-based web application deployed on cloud (Render)
  • Scalability: Handles multiple districts and crime categories dynamically
  • Use Case: Crime risk analysis, district comparison, and safety planning
